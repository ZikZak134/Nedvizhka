# Правило: Адаптивный Дизайн (Tilda-style)

## Философия

Проект использует **подход Tilda** — десктоп и мобильная версии рассматриваются как **два разных интерфейса**, а не просто "сжатие" одного в другой.

## Breakpoints

| Устройство | Ширина | CSS Media Query |
|------------|--------|-----------------|
| **Mobile** | < 768px | `@media (max-width: 767px)` |
| **Tablet** | 768px – 1023px | `@media (min-width: 768px) and (max-width: 1023px)` |
| **Desktop** | ≥ 1024px | `@media (min-width: 1024px)` |

## Ключевые принципы

### 1. Мобильные формы — всегда 1 колонка
```css
/* ❌ НЕПРАВИЛЬНО — 3 колонки на мобильном */
.formGrid {
  display: grid;
  grid-template-columns: repeat(3, 1fr); /* Выезжает за экран! */
}

/* ✅ ПРАВИЛЬНО — 1 колонка на мобильном */
@media (max-width: 767px) {
  .formGrid {
    grid-template-columns: 1fr; /* Всегда одна колонка */
  }
}
```

### 2. Текст не должен обрезаться
```css
/* ✅ Обязательно для заголовков на мобильном */
@media (max-width: 767px) {
  .pageTitle {
    font-size: 1.5rem;
    word-break: break-word;
    overflow-wrap: break-word;
  }
}
```

### 3. Touch-таргеты ≥ 44px
```css
/* ✅ Минимальный размер кликабельных элементов */
@media (max-width: 767px) {
  .formInput, .actionButton {
    min-height: 44px;
    padding: 12px 16px;
  }
}
```

### 4. Скрытие сложных элементов
Некоторые компоненты (многоколоночные таблицы, сложные фильтры) на мобильном **заменяются** на упрощённые версии, а не просто сжимаются.

## Правило для редактирования

> **ВАЖНО**: При изменении дизайна на Desktop проверять, нужно ли также адаптировать Mobile-версию!

### Контрольный список
- [ ] Добавил Desktop-стили? → Проверь Mobile
- [ ] Новый грид/таблица? → 1 колонка на Mobile
- [ ] Новый текстовый блок? → Проверь word-break
- [ ] Новая кнопка/инпут? → Проверь min-height: 44px

## Файловая структура

Мобильные стили хранятся в **том же файле**, но в конце, внутри медиа-запроса:
```css
/* === Desktop стили (по умолчанию) === */
.component { ... }

/* === Mobile стили === */
@media (max-width: 767px) {
  .component { ... }
}
```
